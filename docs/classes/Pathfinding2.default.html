Class default

Directions: top, bottom, left, right (no diagonal fields)

Get the starting node's id (Board.idToArray(ball's position)) Get the end node's position (id of a field cursor's hovering on) Operations on a pair of coordinates -> [y, x] Then it's translated to the correct divs (path coloring) Either this is coloring divs or the 2d array of [y, x] coords is returned and the Board class colors suitable fields

Methods: getOpenLowestF => gets a node with the lowest F cost from the OPEN table checkIfEndNode => checks, if current node is the end one getneighbours => gets all neighbours {INode} of the current node
